✅ 代码简化版（自定义 GCD/LCM）
--------------------

    ll gcd(ll a, ll b) {
        return b ? gcd(b, a % b) : a;
    }
    
    ll lcm(ll a, ll b) {
        return a / gcd(a, b) * b;
    }

* * *

💡 小技巧总结
--------

* GCD 是“向前约束”，LCM 是“向后补齐”。

* 如果两个 GCD 都是某个数的因子，那它们的 LCM 是该数的最小候选。

* 可以反复用 gcd() 去递归恢复构造链是否一致。

==================

* * *

📘 题意回顾
-------

你拥有一个数组 `b`，它是从一个满足「`a[i] | a[i+1]`」的**美丽数组 `a`**，通过将其中若干元素乘以某个整数 `x` 得到的。

### ✅ 结论一：x 必须整除每一项

这就是对 `x` 的**一个下界约束**。所以我们定义：

对所有 `1 ≤ i < n`，取所有 `x_i` 的 **最小公倍数（LCM）**，记为 `res`，就是我们要求的 `x`。

* * *

🔧 构造方案的充分性说明（为什么这就是解）
----------------------

分析四种情况：

| 情况        | 表达式                              | 是否符合构造           |
| --------- | -------------------------------- | ---------------- |
| 两者都乘了 `x` | `b[i]=a[i]*x`, `b[i+1]=a[i+1]*x` | ✅ 没问题            |
| 都没乘       | `b[i]=a[i]`, `b[i+1]=a[i+1]`     | ✅ 没问题            |
| 一个乘，一个不乘  | `b[i]=a[i]*x`, `b[i+1]=a[i+1]`   | ✅ 满足 lcm 条件      |
| 反过来乘      | `b[i]=a[i]`, `b[i+1]=a[i+1]*x`   | ✅ 仍然满足，只要求 x 足够大 |

因此：只要我们取 `x = lcm(b[i]/gcd(b[i], b[i+1]))`，就能覆盖所有可能情况。

✅ 最终代码参考（C++）
-------------

    #include <bits/stdc++.h>
    using namespace std;
    using ll = long long;
    
    ll gcd(ll a, ll b) {
        return b ? gcd(b, a % b) : a;
    }
    
    ll lcm(ll a, ll b) {
        return a / gcd(a, b) * b;
    }
    
    void solve() {
        int n;
        cin >> n;
        vector<ll> b(n);
        for (auto &x : b) cin >> x;
    
        ll x = 1;
        for (int i = 0; i + 1 < n; ++i) {
            ll g = gcd(b[i], b[i+1]);
            ll val = b[i] / g;
            x = lcm(x, val);
        }
    
        cout << x << "\n";
    }
    
    int main() {
        int t;
        cin >> t;
        while (t--) solve();
    }
